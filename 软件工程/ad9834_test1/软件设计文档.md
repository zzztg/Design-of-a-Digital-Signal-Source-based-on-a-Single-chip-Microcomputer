## 1. 系统概述

### 1.1 功能定义

双通道独立可调的数字信号源，支持：

- **频率**：1 Hz ~ 10 MHz，步进 1 Hz ~ 1 MHz（按位编辑）
- **幅值**：±18 V（用 mV 表示），步进 10 mV ~ 10 V
- **相位**：0.0° ~ 360.0°，精度 0.1°
- **波形**：正弦波、三角波、方波
- **通道模式**：单独调 A、单独调 B、同时调 AB

### 1.2 硬件拓扑

```
                        ┌─ AD9834_A (CS_A = PB12)
SPI2 (共享总线) ────────┼─ AD9834_B (CS_B = PB11)
                        └─ DAC      (CS_DAC = PB10)
                               ↑ 幅值控制（待实现）

SYNC_RESET (PB14) ──────── 两片 AD9834 的 RESET 引脚（共用）
```

AD9834 两片共用 SPI2 总线，通过各自的 CS 片选区分。  
SYNC_RESET 引脚同时接两片 AD9834 的硬件 RESET，用于同步启动。

### 3.1 分层结构

```
┌──────────────────────────────────────┐
│            main.c (主循环)            │  ← 应用层

│  ┌────▼──────────▼──────────▼─────┐   │
│  │       UpdateDDS / UpdateAmp     │   │  ← 硬件更新层
│  └────┬──────────┬──────────┬─────┘   │
├───────┼──────────┼──────────┼─────────┤
│  ┌────▼────┐ ┌───▼────┐ ┌──▼──────┐  │  ← BSP/驱动层
│  │ AD9834  │ │  DAC   │ │   HAL   │  │

│  └─────────┘ └────────┘ └─────────┘  │
└──────────────────────────────────────┘
```

### 3.2 设计思想

**数据驱动的脏标记机制**：

```
旋钮/按键  →  修改 g_state (内存)  →  置脏标记
                                          ↓
主循环 Step5  →  检查脏标记  →  批量同步到硬件
```

- 人机交互只修改内存中的 `g_state`，不直接操作硬件
- 脏标记记录哪些参数被修改过
- 每轮循环末尾统一同步硬件，避免频繁 SPI 通信

## 5. 模块详细设计

### 5.1 编码器读取 (`Encoder_ReadDelta`)

**原理**：定时器配置为编码器模式后，计数器会随旋钮转动自动增减。

```
输入: encoder_t *enc
输出: +1(正转一格), -1(反转一格), 0(无转动)

流程:
  1. current = TIM->CNT            // 读当前计数值
  2. raw = current - last_counter  // 计算差值
  3. 更新 last_counter = current
  4. 16位回绕修正 (±65536)
  5. 死区处理: |raw| < 2 → 0      // 过滤抖动
  6. 归一化: raw > 0 → +1, else → -1
```

### 5.2 按键消抖 (`Button_Scan`)

使用三态状态机，每轮主循环调用一次。

```
状态图:

     ┌──────────────────────────────────────┐
     │                                       │
     ▼    (检测到按下)     (10ms后仍按下)     │
  IDLE ──────────────→ DEBOUNCE ──────────→ PRESSED
     ↑                    │                    │
     │    (抖动释放)       │                    │  (检测到释放)
     └────────────────────┘                    │
              (回到 IDLE)                      │
                                               ▼
                                          触发 EVT_PRESS
                                          回到 IDLE
```

**防抖动设计**：进入 DEBOUNCE 后记录当前 tick，满 10ms 再确认。期间如果电平变高（释放），认为是抖动，退回 IDLE。

### 5.3 按位编辑 (`AdjustFrequency/Amplitude/Phase`)

**按位编辑概念**：每个参数有多个"位"（digit），旋钮每转一格，只改变选中位的值。

```
频率 digit 与步进对应：
  digit 0 → 1 Hz       digit 4 → 10 kHz
  digit 1 → 10 Hz      digit 5 → 100 kHz
  digit 2 → 100 Hz     digit 6 → 1 MHz
  digit 3 → 1 kHz

示例：
  当前频率 = 1,234,567 Hz  digit = 3 (kHz位)  正转 1 格
  → 1,234,567 + 1,000 = 1,235,567 Hz

  当前频率 = 1,999,999 Hz  digit = 5 (100kHz位)  正转 1 格
  → 1,999,999 + 100,000 = 2,099,999 Hz  (自动进位)
```

**幅值 digit**：0=10mV, 1=100mV, 2=1V, 3=10V  
**相位 digit**：0=0.1°, 1=1°, 2=10°, 3=100°

每次调整后做限幅检查，防止越界。

### 5.4 通道屏蔽机制

主循环中通过比特位掩码决定哪些通道响应调节：

```c
// selected_ch → 掩码
BOTH (2) → 掩码 = 0x03   (bit0=A路, bit1=B路)
CH_A (0) → 掩码 = 0x01   (仅 A 路)
CH_B (1) → 掩码 = 0x02   (仅 B 路)

if (mask & 0x01) { /* 调 A 路的参数 */ }
if (mask & 0x02) { /* 调 B 路的参数 */ }
```

### 5.5 DAC 幅值控制（伪代码）

当前为占位注释，待具体 DAC 型号确定后实现：

```
幅值映射: 0~18000mV → 12bit DAC 值 0~4095
  dac_value = abs(amplitude_mv) × 4095 / 18000

通信方式: 共用 SPI2，片选 CS_DAC (PB10)

极性控制: 振幅为负时切换 PGA 极性引脚
  POL_x = (amplitude_mv < 0) ? SET : RESET
```

## 6. 主循环流程

```
系统启动
  │
  ├─ HAL_Init()
  ├─ SystemClock_Config()         // 168MHz
  ├─ MX_GPIO_Init()               // GPIO 初始化
  ├─ MX_SPI2_Init()               // SPI 初始化 (21 MBit/s)
  ├─ MX_USART1_UART_Init()        // 串口 (printf)
  ├─ MX_TIM1_Init() ~ MX_TIM3_Init()  // 编码器定时器
  │
  ├─ AD9834_Init()                // 两片 AD9834
  ├─ 启动 TIM 编码器模式
  ├─ 设置系统初始状态
  ├─ UpdateDDS()                  // 第一次同步写入
  │
  └─ while(1) 主循环 ──────────────────────┐
       │                                     │
       ├─ Step1: 读三个编码器增量             │
       │   df = Encoder_ReadDelta(g_enc_freq)│
       │   da = Encoder_ReadDelta(g_enc_amp) │
       │   dp = Encoder_ReadDelta(g_enc_phase)│
       │                                     │
       ├─ Step2: 编码器按键 → 切换编辑位       │
       │   digit_idx[i] = (digit_idx[i]+1)%N  │
       │                                     │
       ├─ Step3: 独立按键                    │
       │   KEY_SW1 → 切换通道 (A/B/BOTH)     │
       │   KEY_SW2 → 切换波形 (Sin/Tri/Sqr)  │
       │                                     │
       ├─ Step4: 旋转 → 调参                 │
       │   if (mask & ch) AdjustParam(...)   │
       │   同时置相应的脏标记                  │
       │                                     │
       ├─ Step5: 硬件同步                    │
       │   UpdateDDS(&g_state)              │
       │   UpdateAmplitude(&g_state)         │
       │                                     │
       └─ HAL_Delay(10) ← 每 10ms 一圈        │
         (约 100 Hz 轮询频率)                  │
                                              │
         └────────────────────────────────────┘
```

## 7. 双路同步机制

### 7.1 为什么需要同步

两片 AD9834 各自独立工作。如果没有同步机制，各自从随机相位开始输出，两路信号之间会有不确定的相位差。

### 7.2 同步方法

利用 AD9834 的 **硬件 RESET 引脚**（两片共用 SYNC_RESET）：

```
时间轴:

  SYNC_RESET ────────┬── 高电平 ──┬── 低电平 ──────────
                     │            │
  AD9834_A 状态    停止(相位=0)  开始输出
  AD9834_B 状态    停止(相位=0)  开始输出 (同时!)
                     │            │
                     │  在此区间  │
                     │  写入频率  │
                     │  写入相位  │
                     │  写入波形  │
```

### 7.3 代码实现

```c
void UpdateDDS(system_state_t *state)
{
    // 没有任何更新需要 → 跳过
    if (!state->flags.freq_dirty && !state->flags.phase_dirty && !state->flags.wave_dirty)
        return;

    // Step 1: 拉高 SYNC_RESET，两片同时停止，相位累积器归零
    HAL_GPIO_WritePin(SYNC_RESET_GPIO_Port, SYNC_RESET_Pin, GPIO_PIN_SET);

    // Step 2: 通过 SPI 分别写入两片 AD9834
    if (state->flags.wave_dirty)  { /* 写波形到 A 和 B */ }
    if (state->flags.freq_dirty)  { /* 写频率到 A 和 B */ }
    if (state->flags.phase_dirty) { /* 写相位到 A 和 B */ }

    // Step 3: 拉低 SYNC_RESET，两片同时从相位 0 开始输出
    HAL_GPIO_WritePin(SYNC_RESET_GPIO_Port, SYNC_RESET_Pin, GPIO_PIN_RESET);
}
```

### 7.4 同步时机

仅在频率或相位发生变化时触发同步。波形切换不影响相位累积器，但为了实现统一管理，当前实现在任意 DDS 参数变化时都做同步。
